# 管理端操作日志增强设计

**日期：** 2026-05-09
**版本：** v1.0

---

## 一、需求概述

增强管理端操作日志模块，以清晰展示管理员操作日志为核心，AI智能分析为辅助功能。

### 核心定位

- **主体**：管理员操作日志列表，清晰明了
- **辅助**：AI智能分析（管理员主动触发）

### 功能范围

| 功能 | 说明 |
|-----|------|
| 日志列表 | 核心展示，分页、筛选 |
| 增强展示 | 信息层次、可读性、视觉、快捷筛选 |
| AI分析 | 管理员主动触发，结构化卡片展示结果 |

---

## 二、日志列表增强

### 2.1 信息层次

**表格列顺序（左→右）：**

| 列 | 宽度 | 说明 |
|---|-----|------|
| 操作时间 | 160px | 左固定，最重要 |
| 管理员 | 120px | 头像+昵称 |
| 操作类型 | 120px | 彩色标签 |
| 操作详情 | min-width:200px | 格式化显示 |
| IP地址 | 130px | 右固定 |

### 2.2 可读性增强

**操作详情格式化：**
- JSON对象转为可读文本：`禁用用户 "小明"，原因：违规发言`
- 对象类型前缀：`用户 #123`、`日志`、`仪表盘`

**时间格式：**
- 当天：`今天 14:32:15`
- 昨天：`昨天 09:15:00`
- 更早：`05-08 10:30`

### 2.3 视觉强化

**操作类型标签样式：**

| 操作 | 颜色 | 图标 |
|-----|------|-----|
| user.disable | 红色 #EF4444 | ● |
| user.enable | 绿色 #059669 | ● |
| user.export | 蓝色 #0284C7 | ● |
| login/logout | 灰色 #64748B | ● |
| 其他 | 青色 #0891B2 | ● |

**异常操作行样式：**
- 禁用/启用操作行左侧加4px彩色边框
- 整行hover时微微高亮

### 2.4 快捷筛选

**工具栏快捷按钮：**

```
[全部] [禁用] [启用] [导出] [登录登出]
```

点击即切换筛选条件，无需下拉选择。

## 三、AI智能分析

### 3.1 触发方式

- 管理员在日志列表页点击「AI智能分析」按钮
- 管理员主动触发，非定时

### 3.2 数据范围

- 读取最近7天日志
- 聚合统计数据（不传原始日志，控制成本）

### 3.3 分析内容

**1. 操作概览卡片：**
- 总操作次数
- TOP3操作类型及次数
- 操作高峰时段（几点）

**2. 7天趋势图：**
- 折线图展示每天操作量
- X轴：日期，Y轴：操作次数

### 3.4 交互流程

```
点击「AI分析」按钮 → 显示loading → 调用星火API → 展示结构化卡片
```

### 3.5 API设计

**请求：**
```
POST /api/admin/logs/analyze
```

**响应：**
```json
{
  "success": true,
  "data": {
    "summary": "近7天共进行45次操作，主要集中在下午2-4点...",
    "total_count": 45,
    "top_actions": [
      {"action": "user.disable", "count": 12},
      {"action": "user.enable", "count": 8},
      {"action": "user.view", "count": 15}
    ],
    "peak_hour": 14,
    "trend": [
      {"date": "2026-05-03", "count": 5},
      {"date": "2026-05-04", "count": 8},
      ...
    ]
  }
}
```
