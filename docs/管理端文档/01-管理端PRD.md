# AI小商 - 管理端产品需求文档（PRD）

> 管理端定位：智慧校园AI助手的运营管理后台，服务于学校管理员/运营人员。

---

## 一、产品概述

### 1.1 产品定位

**"AI小商智能运营中心"**

管理端是AI小商智慧校园AI助手的运营管理后台，通过数据可视化、用户管理、操作审计等功能，帮助学校管理员高效运营和维护系统。

### 1.2 核心价值

| 价值点 | 说明 |
|-------|------|
| 数据驱动 | 通过数据驾驶舱实时掌握系统运营状况 |
| 高效管理 | 集中管理用户、权限、内容，降低运营成本 |
| 安全合规 | 操作日志审计，数据可追溯 |
| 智能预警 | API额度预警、异常行为预警，保障系统稳定 |

### 1.3 目标用户

**管理员用户画像：**
- 学校信息中心/教务处老师
- 负责维护AI小商系统
- 需要查看数据、统计数据、管理用户
- 通常使用PC端访问

### 1.4 管理员说明

管理端使用单一管理员账号，统一管理所有功能。

**默认管理员：**
- 用户名：`admin`
- 密码：`123456`

### 3.2 用户管理中心

#### 3.2.1 用户列表

**展示字段：**
| 字段 | 说明 | 备注 |
|-----|------|------|
| ID | 用户ID | |
| 昵称 | 用户昵称 | |
| 手机号 | 登录手机号 | 脱敏显示 |
| 角色 | 学生/教师 | |
| 专业/院系 | 所属 | 学生显示专业，教师显示院系 |
| 状态 | 正常/禁用 | |
| 注册时间 | 账号创建时间 | |
| 最后登录 | 最近一次登录时间 | |
| 对话次数 | AI对话次数 | |

**筛选条件：**
- 角色：全部/学生/教师
- 状态：全部/正常/禁用
- 关键词：昵称/手机号搜索

#### 3.2.2 账户管理

**禁用用户：**
- 用户无法登录
- 显示禁用原因（可选填）
- 记录操作日志

**启用用户：**
- 恢复用户登录权限

#### 3.2.3 导出功能

**导出格式：** Excel (.xlsx)

**导出字段：**
- ID、昵称、手机号、角色、专业/院系、状态、注册时间、最后登录、对话次数

**文件名格式：**
```
AI小商用户列表_2026-05-08.xlsx
```

### 3.3 操作日志

#### 3.3.1 日志内容

| 字段 | 说明 |
|-----|------|
| 操作时间 | 精确到秒 |
| 管理员 | 管理员昵称 |
| 操作类型 | 如：禁用用户、导出数据、查看用户 |
| 操作详情 | 具体操作内容 |
| IP地址 | 管理员IP |

#### 3.3.2 操作类型定义

| 操作类型 | 说明 |
|---------|------|
| user.disable | 禁用用户 |
| user.enable | 启用用户 |
| user.export | 导出用户数据 |
| user.view | 查看用户详情 |
| log.view | 查看操作日志 |
| dashboard.view | 查看数据看板 |

#### 3.3.3 查询条件

- 时间范围：开始日期 ~ 结束日期
- 操作类型：下拉选择
- 管理员：下拉选择
- 关键词：搜索操作详情

## 五、非功能需求

### 5.1 性能指标

| 指标 | 要求 |
|-----|-----|
| 页面加载 | ≤2秒 |
| 数据刷新 | ≤3秒 |
| Excel导出 | ≤10秒（10000条数据） |
| 并发 | 支持≥10管理员同时在线 |

### 5.2 安全需求

| 需求 | 说明 |
|-----|-----|
| 权限控制 | 管理员需登录，不同角色不同权限 |
| 操作审计 | 所有敏感操作记录日志 |
| 数据脱敏 | 手机号等敏感信息脱敏展示 |

### 5.3 数据刷新

| 数据类型 | 刷新频率 | 说明 |
|---------|---------|------|
| 核心指标 | 30秒 | 今日活跃、API调用等 |
| 功能排行 | 1分钟 | |
| 趋势图 | 5分钟 | |
| 用户列表 | 手动刷新 | |
