# AI小商 - 管理端接口规范

> 本文档定义管理端所有API的请求/响应格式。

---

## 一、接口基础规范

### 1.1 认证方式

管理端使用独立的JWT认证，与学生/教师端分开。

```
Authorization: Bearer <admin_token>
```

### 1.2 统一响应格式

**成功响应：**
```json
{
  "success": true,
  "data": { ... },
  "message": "操作成功"
}
```

**错误响应：**
```json
{
  "success": false,
  "error": {
    "code": "ERROR_CODE",
    "message": "错误描述"
  }
}
```

### 1.3 错误码定义

| 错误码 | 说明 |
|-------|------|
| UNAUTHORIZED | 未登录或token过期 |
| FORBIDDEN | 无权限访问 |
| NOT_FOUND | 资源不存在 |
| VALIDATION_ERROR | 参数校验失败 |
| INTERNAL_ERROR | 服务器内部错误 |

- Content-Disposition: attachment; filename="AI小商用户列表_2026-05-08.xlsx"

| end_date | string | 否 | 结束日期（YYYY-MM-DD） |
| action | string | 否 | 操作类型 |

## 六、字段说明

### 6.1 user.role 枚举

| 值 | 说明 |
|----|------|
| student | 学生 |
| teacher | 教师 |

### 6.2 user.status 枚举

| 值 | 说明 |
|----|------|
| active | 正常 |
| disabled | 已禁用 |

### 6.3 admin 响应字段

| 字段 | 类型 | 说明 |
|-----|------|------|
| id | int | 管理员ID |
| username | string | 用户名 |
| nickname | string | 昵称 |
| created_at | string | 创建时间 |

### 6.4 log.action 枚举

| 值 | 说明 |
|----|------|
| user.disable | 禁用用户 |
| user.enable | 启用用户 |
| user.export | 导出用户数据 |
| user.view | 查看用户详情 |
| log.view | 查看操作日志 |
| dashboard.view | 查看数据看板 |

### 6.6 api_name 枚举（API接口）

| 值 | 说明 |
|----|------|
