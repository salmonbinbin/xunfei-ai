### 1. 课表识别 - 地点识别失败

**问题描述**：
- 图片理解API返回的结果中，所有课程的`location`字段均为`null`
- 模型完全无法识别课表中的地点信息（如"九教501"、"八教505"等）

**根本原因**：
- Prompt过于复杂，混淆信息过多
- 模型被错误的信息干扰，无法正确识别

**解决方案**：
1. 简化 `timetable.py` 中的图片理解prompt
2. 简化 `xinghuo_service.py` 中的 `TIMETABLE_PARSE_PROMPT`
3. 移除所有关于"老师位于地点后面"的说明
4. 移除teacher字段（用户决定不显示老师）
5. 简化示例，专注识别：课程名、地点、节次、周数

**修改文件**：
- `backend/app/routers/timetable.py` - 简化image_understanding的prompt

### 5. AI学习助手 - Pydantic验证懒加载错误

**问题描述**：
- 点击课程卡片查看详情时报500错误
- 日志显示：`greenlet_spawn has not been called; can't call await_only() here`
- Pydantic在验证`CourseWithAIResponse`时尝试访问`ai_insight`关系属性

**根本原因**：
- SQLAlchemy的懒加载在异步上下文中无法工作
- `model_validate(course)`时触发了懒加载的`ai_insight`关系

**解决方案**：
1. 在`get_course_detail`接口中不使用ORM的懒加载关系
2. 手动构建响应数据字典，避免访问`ai_insight`关系
3. 分别查询Course和CourseAIInsight

**修改文件**：
- `backend/app/routers/timetable.py` - 修改`get_course_detail`函数，手动构建响应数据

```python
# 手动构建响应，避免懒加载问题
response_data = {
    "id": course.id,
    "user_id": course.user_id,
    "name": course.name,
    "code": course.code,
    "credit": float(course.credit) if course.credit else None,
    "category": course.category,
    "day_of_week": course.day_of_week,
    "start_slot": course.start_slot,
    "end_slot": course.end_slot,
    "week_range": course.week_range,
    "location": course.location,
    "teacher": course.teacher,
    "ai_tip": course.ai_tip,
    "is_active": course.is_active,
    "created_at": course.created_at,
    "ai_insight": None
}

if insight:
    from app.schemas.timetable import AIInsightResponse
    response_data["ai_insight"] = AIInsightResponse.model_validate(insight)

return CourseWithAIResponse(**response_data)
```

## 三、识别Prompt优化记录

### 优化前的Prompt（失败）

过于复杂，包含大量混淆信息：
- 强调"老师位于地点后面"
- 包含多种格式示例
- teacher字段要求复杂

### 优化后的Prompt（成功）

```markdown
你是一个专业的课表识别助手。请从课表图片中提取课程信息。

返回严格JSON格式（**只返回JSON，不要其他文字！**）：
{
  "courses": [
    {
      "name": "课程名称（纯净，不含其他信息）",
      "location": "如九教501、八教505、实训楼402，看不到写null",
      "day_of_week": 1-7数字（1=周一）,
      "start_slot": 1-12数字,
      "end_slot": 1-12数字,
      "week_range": "如1-16周、1-17周，看不到写null"
    }
  ]
}

**识别示例：**
- 图片文字"高等数学 九教501 1-2节 1-16周" → name:高等数学, location:九教501, day_of_week:1, start_slot:1, end_slot:2, week_range:1-16周
- 图片文字"大学英语 实训楼402 3-4节" → name:大学英语, location:实训楼402, day_of_week:2, start_slot:3, end_slot:4, week_range:null

**地点格式**：必须是"教学楼+数字"，如九教501、八教505、实训楼402。不是"教室501"也不是"501"。

**节次格式**：如"1-2节"、"3-4节"，对应start_slot=1, end_slot=2
```

**经验总结**：
- Prompt越简单越好
- 不要在一个prompt里混合太多概念
- 示例要清晰、具体
- 明确指出格式要求而不是描述复杂规则

## 五、技术要点

### 1. SQLAlchemy异步处理
在异步FastAPI中使用SQLAlchemy时，避免懒加载：
- 不要直接传递ORM对象给Pydantic的`model_validate()`
- 手动提取字段构建字典
- 必要时使用`selectinload`预加载

### 2. Prompt设计原则
- 简洁明了
- 格式示例清晰
- 避免歧义
- 逐步引导而非一次性给出所有信息

### 3. 数据库迁移
当模型添加新字段后，需要手动执行ALTER TABLE：
```sql
ALTER TABLE table_name ADD COLUMN column_name TYPE DEFAULT default_value;
```
