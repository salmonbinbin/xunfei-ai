## 三、遇到的问题及解决

### 3.1 Logger 名称不一致

**问题**: `activity.py` 中使用 `logging.getLogger("api")` 而非 `logging.getLogger("activity")`

**解决**: 修改为正确的 logger 名称，符合 CLAUDE.md 中按模块命名 logger 的规范

### 3.2 CSS 语法错误

**问题**: PostCSS 解析报错 `Unclosed bracket`，原因是多个选择器逗号分隔时换行格式错误

**错误代码**:
```css
.form-input-number :deep(.el-input-number__decrease,
.form-input-number :deep(.el-input-number__increase) {  /* 缺少右括号 */
```

**解决**: 修正为正确的多选择器格式
```css
.form-input-number :deep(.el-input-number__decrease),
.form-input-number :deep(.el-input-number__increase) {
```

### 3.3 Pydantic extra 配置不生效

**问题**: `extra = "ignore"` 使用 class-based Config 在新版 Pydantic 中不生效，导致 `FEISHU_WEBHOOK_STUDENT_UNION` 被拒绝

**解决**: 改用 `model_config = SettingsConfigDict(...)` 方式

### 3.4 飞书 Webhook 环境变量未加载

**问题**: `FeishuService` 在 Uvicorn 启动时初始化，`load_dotenv('.env')` 尚未调用，导致 webhooks 字典为空

**日志表现**:
```
[Feishu] FEISHU_WEBHOOK_STUDENT_UNION not set for group: student_union
[Feishu] 未找到群组配置: student_union
```

**解决**:
1. 修改 `FeishuService` 实现延迟加载（`_ensure_webhooks_loaded()`）
2. 在 `_load_webhooks_from_env()` 中显式调用 `load_dotenv('.env')`
3. 使用单例模式避免重复初始化

## 四、经验总结

### 4.1 环境变量加载时机

- **问题根源**: Python 的 `os.environ` 在模块导入时就已固定，后续修改 `.env` 文件不会影响已导入的变量
- **解决方案**: 在需要使用环境变量的函数/方法内部显式调用 `load_dotenv('.env')`，确保读取最新配置
- **推荐模式**: 延迟加载 + 显式加载结合

### 4.2 Pydantic Settings V2 迁移

- 新版 Pydantic 使用 `SettingsConfigDict` 替代 class-based `Config`
- `extra = "ignore"` 需要写在 `SettingsConfigDict` 中，而非 `Config` 类
- 环境变量读取优先顺序：命令行参数 > 环境变量 > .env 文件 > 默认值

### 4.3 CSS 选择器换行规范

- 多选择器逗号分隔时，每个选择器单独一行，提升可读性
- `:deep()` 伪类用于穿透 Vue Scoped CSS

### 4.4 前端 UI 风格统一

- 项目采用「清新校园风」设计语言：青色主色调 `#0891B2`、浅灰背景 `#F8FAFC`、白色卡片
- 新增功能页面应继承统一风格，而非独立设计
- 使用 Element Plus 组件保持一致性

### 4.5 飞书 Webhook 配置检查清单

1. 在飞书群中添加自定义机器人
2. 复制机器人的 Webhook URL
3. 填入 `.env` 对应变量
4. 重启后端服务
5. 验证日志确认 webhook 加载成功
