## 二、问题与解决方案

### 问题1：语音输入 422 错误

**现象**：`POST /api/chat/voice` 返回 422 Unprocessable Entity

**原因**：
1. 后端参数类型注解问题：`audio: UploadFile = File(...)` 写法有误
2. 前端发送的 formData 字段名与后端不匹配

**解决**：
```python
# 修改前
@router.post("/voice")
async def chat_voice(file: UploadFile = File(...)):

# 修改后
@router.post("/voice")
async def chat_voice(audio: UploadFile = File(...)):
```

前端 formData 使用 `formData.append('audio', ...)` 与后端参数名一致。

### 问题2：语音输入 500 错误

**现象**：参数修正后仍然返回 500 Internal Server Error

**原因**：`handle_app_errors` 装饰器拦截了 HTTPException 并错误转换为 500

**解决**：
```python
@handle_app_errors
async def wrapper(*args, **kwargs):
    try:
        return await func(*args, **kwargs)
    except HTTPException:
        raise  # 直接抛出，不拦截
    except AppException as e:
        raise HTTPException(status_code=e.status_code, detail=e.to_dict())
```

### 问题3：TTS音频 404 无法播放

**现象**：TTS 合成的音频返回 URL，但前端无法加载播放

**原因**：前端开发服务器（5173）无法访问后端 `/uploads/` 下的静态文件

**解决**：在 `vite.config.js` 添加代理
```javascript
server: {
  proxy: {
    '/api': { target: 'http://localhost:8000', changeOrigin: true },
    '/uploads': { target: 'http://localhost:8000', changeOrigin: true }
  }
}
```

### 问题4：iFlytek ASR 状态 -1 错误

**现象**：ASR 轮询返回 status=-1，无法正确处理

**原因**：-1 表示"已提交，等待处理"，但代码未处理此状态

**解决**：将 -1 视为继续轮询的条件
```python
elif process_status == -1:
    self.logger.debug("[ASR] Transcription submitted, waiting...")
    retry_count += 1
    continue
```

### 问题5：iFlytek API 限流（100012）

**现象**：`access too fast, please wait a moment`

**原因**：iFlytek ASR 有调用频率限制

**解决**：这是正常行为，提醒用户每次语音输入后等待 3-5 秒再进行下一次

### 问题6：TTS host 签名错误

**现象**：TTS WebSocket 签名验证失败

**原因**：签名中使用的 host 与实际请求地址不匹配

**解决**：
```python
signature_origin = (
    f"host: tts-api.xfyun.cn\n"  # 之前错误写成 ws-api.xfyun.cn
    f"date: {date}\n"
    f"GET /v2/tts HTTP/1.1"
)
```

## 三、经验总结

### 3.1 API 命名一致性
- 前后端参数命名必须保持一致
- 建议在接口文档中明确定义每个字段名

### 3.2 错误处理分层
- HTTP 异常（HTTPException）直接抛出
- 业务异常（AppException）统一转换格式
- 第三方异常单独处理并记录日志

### 3.3 前端音频处理
- 使用原生 `Audio` 对象而非第三方库
- 通过 `pause()/play()` 实现暂停/继续
- 在 `onended` 回调中清理状态

### 3.4 Vite 开发服务器代理
- 静态资源文件需要配置代理才能在开发环境访问
- 生产环境需要 nginx 配置相同的静态文件路由

### 3.5 讯飞 API 注意事项
- ASR 语音识别有频率限制，客户端需控制调用间隔
- TTS 发音人参数需要查阅讯飞文档确认有效值
- WebSocket 签名 host 必须与实际请求 host 完全匹配
