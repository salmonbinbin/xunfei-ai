# 2026-05-09 数据驾驶舱UI优化与用户统计功能工作日志

## 一、工作概述

本次工作针对管理端数据驾驶舱进行UI布局优化，将中间位置的卡片从"学生登录数/教师登录数"改为"学生用户人数/教师用户人数"，直接显示数据库中学生用户和教师用户的总数。

## 三、技术实现

### 3.1 后端修改

**文件：** `app/routers/admin_console/dashboard.py`

**新增统计字段：**

```python
# 学生用户总数（来自users表）
result = await db.execute(
    select(func.count(User.id))
    .where(User.role == "student", User.is_deleted == 0)
)
total_students = result.scalar() or 0

# 教师用户总数（来自users表）
result = await db.execute(
    select(func.count(User.id))
    .where(User.role == "teacher", User.is_deleted == 0)
)
total_teachers = result.scalar() or 0
```

**返回数据新增：**
```python
"total_students": total_students,
"total_teachers": total_teachers
```

**数据来源说明：**

| 字段 | 数据来源 | 说明 |
|-----|---------|------|
| `total_students` | `users` 表 | `role='student'` 且 `is_deleted=0` 的用户总数 |
| `total_teachers` | `users` 表 | `role='teacher'` 且 `is_deleted=0` 的用户总数 |

## 五、关于数据显示的说明

### 5.1 为什么"功能使用排行"只有3个模块？

当前 `user_logs` 表中的记录：

| 模块 | 记录数 |
|-----|-------|
| ai_sister | 1条 |
| course_advisor | 2条 |
| activity | 1条 |

**原因：** 只有这些功能被实际使用过，其他功能（录音回顾、课表管理等）还没有产生操作日志。

### 5.2 为什么"API调用统计"只显示星火大模型？

当前 `api_logs` 表中的记录：

| API | 记录数 |
|-----|-------|
| spark | 11条 |

**原因：** 目前只有星火大模型（AI对话）被使用过并记录了日志。ASR（语音识别）、TTS（语音合成）、OCR（文字识别）等API虽然代码中已经集成了日志记录功能，但还没有被实际调用过。

**结论：** 数据展示是准确的，反映了真实的使用情况。随着学生端和教师端功能的广泛使用，日志数据会逐渐丰富。
