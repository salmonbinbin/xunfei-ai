## 一、功能概述

智能备课教案生成功能是面向教师的AI辅助备课工具，核心流程为：

1. **创建教案** → 输入标题、课程名、知识点等基本信息
2. **生成大纲** → 调用星火大模型，根据知识点生成结构化教学大纲
3. **生成PPT** → 调用讯飞智能PPT API，将大纲内容填充到PPT模板
4. **轮询状态** → 定时查询PPT生成进度，完成后提供下载链接

## 三、讯飞API使用

### 3.1 星火大模型API（生成大纲）

**API**: `POST https://spark-api-open.xf-yun.com/v1/chat/completions`

**用途**: 生成教学大纲

**调用方式**:
```python
messages = [{"role": "user", "content": prompt}]
response = await xinghuo_service.chat_completion(
    messages=messages,
    user_id="lesson_plan",
    temperature=0.7,
    max_tokens=4096
)
```

**Prompt设计**:
```
请根据以下信息生成一份详细的教学大纲，返回严格的JSON格式。
输入信息：{知识点}
返回格式：
{
    "title": "教案标题",
    "subTitle": "副标题",
    "chapters": [
        {"chapterTitle": "第一章", "chapterContents": ["要点1", "要点2"]}
    ]
}
```

## 四、遇到的问题与解决

### 4.1 问题一：签名算法错误

**现象**: 讯飞API返回"鉴权错误"

**原因**: 签名算法不匹配。讯飞PPT API使用 `MD5(app_id+timestamp) → HMAC-SHA1(api_secret, result) → Base64`

**解决**:
```python
def _generate_signature(self, timestamp: str) -> str:
    text = self.app_id + timestamp
    auth = hashlib.md5(text.encode('utf-8')).hexdigest()
    signature = base64.b64encode(
        hmac.new(
            self.api_secret.encode('utf-8'),
            auth.encode('utf-8'),
            hashlib.sha1
        ).digest()
    ).decode('utf-8')
    return signature
```

### 4.2 问题二：PPT模板图片不显示

**现象**: 模板列表中每个模板的`detailImage`字段返回的是JSON字符串而非图片URL

**原因**: 讯飞API返回的`detailImage`是一个嵌套JSON对象，包含多种尺寸的封面图

**解决**: 解析JSON字符串，提取`titleCoverImage`作为预览图
```python
for record in records:
    detail_image = record.get("detailImage", "")
    if detail_image:
        img_data = json.loads(detail_image)
        record["detailImage"] = img_data.get("titleCoverImage", "")
```

### 4.3 问题三：模板名称显示"未命名模板"

**现象**: 所有模板都显示"未命名模板"

**原因**: API返回的模板数据中没有`title`字段，只有`style`、`industry`、`color`等属性

**解决**: 组合这些属性构造模板名称
```python
record["title"] = f"{style} {industry} {color}"
```

### 4.4 问题四：生成大纲接口报错

**现象**: `generate_outline` 调用讯飞PPT API报错

**原因**: 大纲生成应该使用星火大模型（LLM），而不是讯飞PPT API

**解决**: 重写`generate_outline()`方法，调用`xinghuo_service.chat_completion()`生成大纲

### 4.5 问题五：生成大纲返回内容不是有效JSON

**现象**: LLM返回的内容包含markdown代码块标记，无法解析

**原因**: LLM常返回```json ... ```格式

**解决**: 清理markdown格式
```python
cleaned = re.sub(r'^```json\s*', '', response.strip(), flags=re.IGNORECASE)
cleaned = re.sub(r'\s*```$', '', cleaned.strip())
```

### 4.6 问题六：PPT生成报"content-type错误"

**现象**: 讯飞API返回"请求头content-type错误"

**原因**: `createPptByOutline`接口期望`Content-Type: application/json`，但之前用form-data发送

**解决**: 改用`_http_post_json()`方法发送JSON格式请求

### 4.7 问题七：PPT生成报"请求体body有误"

**现象**: API返回"请求体body有误，请检查body参数是否正确"

**原因**:
1. 缺少API要求的完整字段（PPTCreate.to_dict()需要所有字段）
2. `outline.chapterContents`格式错误：API期望对象数组`[{"chapterTitle": "...", "chapterContents": []}]`，但LLM返回的是字符串数组`["..."]`

**解决**:
1. 补全所有字段（包括null值）
2. 添加`_convert_outline_for_api()`转换大纲格式

### 4.8 问题八：PPT生成的内容是空模板

**现象**: 下载的PPT只有模板样式，没有实际内容

**原因**: `generate-ppt`接口中`outline`参数硬编码为空字典`{}`，生成的大纲内容从未传递

**解决**:
1. 在`LessonPlanGeneratePptRequest`中添加`outline`字段
2. 前端在调用`generatePpt`时传递`outline: outline.value`
3. 路由层将`request.outline`传给服务层

## 六、关键代码片段

### 6.1 大纲生成（使用星火大模型）

```python
async def generate_outline(self, query: str, ...) -> Dict[str, Any]:
    prompt = self._build_outline_prompt(query)
    messages = [{"role": "user", "content": prompt}]
    response = await xinghuo_service.chat_completion(messages=messages, ...)
    outline_data = self._parse_outline_response(response)
    return {"sid": f"outline_{uuid.uuid4().hex[:16]}", "outline": outline_data}
```

### 6.2 PPT生成（讯飞API）

```python
async def generate_ppt_by_outline(self, outline: Dict[str, Any], ...) -> Dict[str, Any]:
    api_outline = self._convert_outline_for_api(outline)  # 转换格式
    data = {
        "query": query,
        "outline": api_outline,
        "author": author,
        "language": language,
        "isFigure": is_figure,
        ...
    }
    result = await self._http_post_json(self.PPT_CREATE_BY_OUTLINE_URL, data)
    return {"sid": result["data"]["sid"], "title": result["data"]["title"], ...}
```

### 6.3 格式转换

```python
def _convert_outline_for_api(self, outline: Dict[str, Any]) -> Dict[str, Any]:
    api_chapters = []
    for chapter in outline.get("chapters", []):
        api_contents = []
        for content in chapter.get("chapterContents", []):
            if isinstance(content, str):
                api_contents.append({"chapterTitle": content, "chapterContents": []})
        api_chapters.append({
            "chapterTitle": chapter.get("chapterTitle", ""),
            "chapterContents": api_contents
        })
    return {"title": outline.get("title"), "chapters": api_chapters}
```

## 七、经验总结

### 7.1 讯飞API使用经验

1. **域名区分**: 不同讯飞产品使用不同域名
   - 星火大模型: `spark-api-open.xf-yun.com`
   - 智能PPT: `zwapi.xfyun.cn`

2. **Content-Type必须匹配**: 讯飞PPT的`createPptByOutline`要求JSON格式，而`createOutline`要求form-data格式

3. **签名算法**: 不同产品可能使用不同的签名算法，PPT API使用MD5+HMAC-SHA1+Base64

4. **字段完整性**: 某些API要求请求体包含完整字段结构，不能只传部分字段

### 7.2 大模型调用经验

1. **Prompt设计**: 明确要求返回JSON格式，并给出具体的JSON结构示例
2. **响应清理**: 大模型常返回markdown包裹的JSON，需要清理后才能解析
3. **格式转换**: 不同模块之间的数据格式可能不同，需要编写专门的转换函数

### 7.3 调试技巧

1. **日志记录**: 在每个API调用点记录请求参数和响应结果
2. **最小化复现**: 用独立的测试脚本复现问题，排除其他干扰
3. **逐步排查**: 先验证签名、Content-Type等基础问题，再排查业务逻辑
