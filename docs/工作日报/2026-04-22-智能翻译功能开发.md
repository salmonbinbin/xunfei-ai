## 一、功能概述

本次工作为 AI小商 项目新增了**智能翻译**功能模块，包含即时文本翻译和文档翻译两大核心功能，面向广州商学院师生提供便捷的中英日韩等多语言翻译服务。

### 1.1 功能需求

| 功能 | 描述 | 目标用户 |
|-----|------|---------|
| 即时翻译 | 输入中文文本，实时翻译为目标语言 | 日常翻译需求 |
| 文档翻译 | 上传 txt/docx 文件，翻译后支持预览和下载 | 文档翻译场景 |
| 多语言支持 | 中文、英文、日文、韩文、法文、德文、西班牙文、俄文、阿拉伯文 | 国际化需求 |

### 3.2 API 接口

| 方法 | 路径 | 功能 |
|-----|------|------|
| POST | `/api/translate` | 即时文本翻译 |
| POST | `/api/translate/docx` | 文档翻译上传 |
| GET | `/api/translate/preview/{task_id}` | 获取翻译预览 |
| GET | `/api/translate/download/{task_id}` | 下载翻译文档 |

### 3.3 讯飞 API 集成

**API 地址**：`https://itrans.xf-yun.com/v1/its`

**鉴权流程**：
1. 生成 RFC1123 格式时间戳
2. 构造签名源：`host: {host}\ndate: {date}\nPOST {path} HTTP/1.1`
3. 使用 HMAC-SHA256 和 API Secret 进行签名
4. 构造 Authorization 头并 base64 编码
5. 将鉴权参数附加到 URL query string

**请求体结构**：
```json
{
  "header": { "app_id": "xxx", "status": 3 },
  "parameter": { "its": { "from": "cn", "to": "en", "result": {} } },
  "payload": { "input_data": { "encoding": "utf8", "status": 3, "text": "base64..." } }
}
```

**响应解析**：
```json
{
  "payload": {
    "result": {
      "text": "base64编码的翻译结果"
    }
  }
}
```

### 3.4 重试机制

```python
MAX_RETRIES = 3
RETRY_DELAYS = [1.0, 2.0, 3.0]  # 递增间隔

async def _translate_with_retry(self, text, source_lang, target_lang):
    for attempt in range(self.MAX_RETRIES):
        try:
            return await self._do_translate(text, source_lang, target_lang)
        except Exception as e:
            if attempt < self.MAX_RETRIES - 1:
                await asyncio.sleep(self.RETRY_DELAYS[attempt])
            else:
                raise ThirdPartyException(...)
```

### 3.5 长文本处理

当文本超过 5000 字符时，自动分片翻译：
1. 优先按段落分割（双换行 `\n\n`）
2. 段落仍过长则按句子分割（中英文句号、问号、感叹号）
3. 分片翻译后拼接结果
4. 单个分片翻译失败时保留原文

## 四、前端实现

### 4.1 API 封装

```javascript
// frontend/src/api/translate.js
import api from '@/api/index'

export function translateText(text, sourceLang, targetLang) {
  const formData = new FormData()
  formData.append('text', text)
  formData.append('source_lang', sourceLang)
  formData.append('target_lang', targetLang)
  return api.post('/translate', formData, {
    headers: { 'Content-Type': 'multipart/form-data' }
  })
}

export function translateDocx(file, targetLang) {
  const formData = new FormData()
  formData.append('file', file)
  formData.append('target_lang', targetLang)
  return api.post('/translate/docx', formData, {
    headers: { 'Content-Type': 'multipart/form-data' }
  })
}
```

## 五、遇到的问题与解决

### 5.1 API 导入路径错误

**问题**：`translate.js` 导入路径错误，引用了不存在的 `@/utils/request`

**错误信息**：
```
Module not found: @/utils/request
```

**解决**：
```javascript
// 错误写法
import api from '@/utils/request'

// 正确写法
import api from '@/api/index'
```

### 5.2 数据库表不存在

**问题**：`translation_tasks` 表未创建

**错误信息**：
```
Table 'ai_xiaoshang.translation_tasks' doesn't exist
```

**解决**：执行 DDL 创建表
```sql
CREATE TABLE translation_tasks (
  id INT AUTO_INCREMENT PRIMARY KEY,
  task_id VARCHAR(64) UNIQUE NOT NULL,
  user_id INT NOT NULL,
  task_type VARCHAR(20) NOT NULL,
  source_lang VARCHAR(10) NOT NULL,
  target_lang VARCHAR(10) NOT NULL,
  original_content TEXT,
  translated_content TEXT,
  status VARCHAR(20) DEFAULT 'pending',
  word_count INT DEFAULT 0,
  error_message TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_task_id (task_id),
  INDEX idx_user_id (user_id),
  INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 5.3 响应数据嵌套访问错误

**问题**：后端返回 `{ success: true, data: { translated_text: "Hello" } }`，前端错误访问 `response.data.translated_text`

**错误现象**：翻译结果显示 `undefined`

**解决**：正确提取嵌套数据
```javascript
// 错误写法
translatedText.value = response.data.translated_text

// 正确写法
const resultData = response?.data?.data || response?.data
translatedText.value = resultData?.translated_text
```

### 5.4 502 Bad Gateway

**问题**：后端服务未运行或崩溃

**解决**：
```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

### 5.5 源语言未固定

**问题**：用户希望源语言固定为中文，但前端仍有语言选择器

**解决**：移除源语言选择器，硬编码源语言为 `cn`
```javascript
// 源语言固定为中文
const response = await translateText(inputText.value, 'cn', targetLang.value)
```

### 5.6 文档翻译目标语言默认值错误

**问题**：`docTargetLang` 初始值为 `'cn'`，导致无法翻译为中文

**解决**：修正默认值为 `'en'`
```javascript
const docTargetLang = ref('en')  // 目标语言默认英文
```

### 5.7 图标风格不一致

**问题**：翻译功能使用的图标复杂度过高，与其他页面简洁风格不统一

**解决**：统一使用简洁箭头图标
```html
<!-- 旧图标（复杂多路径） -->
<path d="M12.87 15.07l-2.54-2.51.03-.03A17.52..."/>

<!-- 新图标（简洁箭头） -->
<path d="M5 12h14M12 5l7 7-7 7"/>
```

## 七、经验总结

### 7.1 讯飞 API 集成经验

1. **鉴权是关键**：讯飞的 HMAC-SHA256 签名鉴权需要严格按官方 demo 实现，任何细微差异都会导致 401 错误
2. **重试机制必要**：网络波动会导致请求失败，递增间隔重试 3 次是最佳实践
3. **长文本分片**：讯飞单次翻译限制 5000 字符，需要实现智能分片
4. **Base64 编码**：文本需要 base64 编码后传输，响应也是 base64 编码

### 7.2 前后端数据交互

1. **响应格式统一**：约定 `{ success: true, data: {...} }` 格式，避免歧义
2. **嵌套层级注意**：Axios 包装后，实际数据在 `response.data.data`，不是 `response.data`
3. **FormData 上传**：文件上传必须使用 `multipart/form-data`，不能用 JSON

### 7.3 前端开发规范

1. **图标风格统一**：同一产品内图标风格必须一致，避免视觉割裂
2. **移动端适配**：预留底部导航空间 (`padding-bottom: 100px`)
3. **错误处理**：所有 API 调用必须 try-catch，并给用户友好提示

### 7.4 数据库设计

1. **索引要建**：常用查询字段（task_id, user_id, status）需要建立索引
2. **软删除考虑**：重要数据建议添加 `is_deleted` 字段支持软删除
3. **事务一致性**：状态变更需要在同一事务内完成
