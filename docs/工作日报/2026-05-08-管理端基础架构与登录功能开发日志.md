## 三、问题与解决

### 3.1 JWT模块导入错误

**问题**：
```
ModuleNotFoundError: No module named 'jwt'
```

**原因**：代码中使用 `import jwt`，但项目使用的是 `python-jose` 库

**解决**：将 `import jwt` 改为 `from jose import jwt`

**涉及文件**：
- `app/routers/admin_console/auth.py`
- `app/routers/admin_console/dashboard.py`
- `app/routers/admin_console/log.py`
- `app/routers/admin_console/user.py`

### 3.2 SQLAlchemy 原始SQL语句错误

**问题**：
```
sqlalchemy.exc.ArgumentError: Textual SQL expression 'SELECT * FROM admins WHER...' should be explicitly declared as text()
```

**原因**：在SQLAlchemy asyncio中使用原始SQL字符串，而不是使用ORM查询

**解决**：将原始SQL改为SQLAlchemy ORM查询

**示例**：
```python
# 错误写法
result = await db.execute(f"SELECT * FROM admins WHERE username = '{request.username}'")

# 正确写法
result = await db.execute(select(Admin).where(Admin.username == request.username))
```

### 3.3 bcrypt密码验证失败

**问题**：使用bcrypt验证默认管理员密码123456失败

**原因**：init_admin_tables.sql中的bcrypt哈希值不正确

**解决**：
```python
import bcrypt
salt = bcrypt.gensalt(rounds=12)
hash = bcrypt.hashpw(password.encode('utf-8'), salt)
# 更新数据库中的密码哈希
```

### 3.4 admin_logs表结构缺失

**问题**：
```
(pymysql.err.OperationalError) (1054, "Unknown column 'is_deleted' in 'field list'")
```

**原因**：数据库中admin_logs表缺少is_deleted和updated_at字段

**解决**：执行ALTER TABLE添加缺失字段
```sql
ALTER TABLE admin_logs ADD COLUMN is_deleted TINYINT DEFAULT 0,
ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
```

## 五、经验总结

### 5.1 技术经验

1. **SQLAlchemy异步ORM**
   - 在FastAPI异步路由中，必须使用`await db.execute()`配合`select()`语句
   - 避免使用原始SQL字符串，改用ORM查询更安全、更易维护
   - 使用`scalar_one_or_none()`获取单个结果，`scalars().all()`获取列表

2. **JWT认证**
   - 项目使用`python-jose`库而非`PyJWT`
   - Token中包含`type: "admin"`区分管理员和普通用户
   - Token有效期7天

3. **bcrypt密码哈希**
   - 密码哈希是单向加密，无法反向解密
   - 验证时使用`bcrypt.checkpw(password.encode(), hash)`
   - rounds参数影响计算成本，默认值12

4. **Pinia状态管理**
   - 使用`defineStore`创建store
   - 可以直接操作store.actions改变状态
   - 配合localStorage实现持久化登录状态

### 5.2 开发流程经验

1. **前后端分离**
   - 后端先完成数据模型和API接口
   - 前端使用Axios封装API调用
   - 前后端通过JSON数据交互

2. **路由守卫**
   - 使用`router.beforeEach`实现页面访问控制
   - 未登录用户重定向到登录页
   - 管理端路由前缀`/admin`

3. **错误处理**
   - 后端使用`try-catch`包裹数据库操作
   - 前端使用`try-catch`配合`ElMessage.error()`展示错误
   - 统一错误响应格式

### 5.3 数据库经验

1. **表结构设计**
   - 使用软删除（is_deleted字段）而非物理删除
   - 添加时间戳字段（created_at, updated_at）
   - 创建必要的索引

2. **SQL初始化**
   - 使用`CREATE TABLE IF NOT EXISTS`避免重复创建
   - 使用`ON DUPLICATE KEY UPDATE`处理重复插入
