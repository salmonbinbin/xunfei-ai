## 一、项目概述

### 1.1 项目背景
AI小商是一个面向广州商学院师生的智慧校园AI助手，已完成学生端开发。本次任务是开发教师端的**智能成绩管理**功能，包括：
- Excel成绩表上传与解析
- AI智能分析报告生成（讯飞星火大模型）
- 成绩数据可视化（ECharts图表）
- 成绩单导出功能

### 1.2 技术栈

| 前端框架 | Vue.js 3 + Vite | 组件化开发 |

| 图表 | ECharts 5.x | 成绩可视化 |

本次开发使用的讯飞技术：

| 技术 | 用途 | 调用方式 |
|-----|------|---------|
| **星火大模型(Spark API)** | 生成AI成绩分析报告 | HTTP API调用 `chat_completion` |

| 讯飞语音识别(ASR) | (学生端已有，教师端本次未使用) | - |

**星火大模型Prompt设计**：
- 根据课程成绩统计数据（平均分、及格率、分数分布、各单项均分）

- 包含：班级概述、高分学生名单、需关注学生、试卷分析（难度/区分度）、教学建议

## 三、遇到的问题与解决

### 3.1 Excel解析问题

**问题1： Worksheet.iter_rows()参数错误**
```
错误信息：Worksheet.iter_rows() got an unexpected keyword argument 'min_rows'. Did you mean 'min_row'?
```
**原因**：openpyxl的iter_rows()方法参数是`min_row`而非`min_rows`（复数形式）
**解决**：修正参数名为`min_row`和`max_row`

**问题2：标题行导致表头识别错误**
```
错误信息：missing_column: student_name
```
**原因**：模拟Excel第1行是标题"2026年春季学期 市场营销 成绩表"，实际表头在第2行
**解决**：添加`_find_header_row()`方法，智能查找包含"学生姓名"和"期末"的行作为表头

**问题3：读取表头时类型错误**
```
错误信息：'str' object has no attribute 'value'
```
**原因**：使用`values_only=True`时返回的是元组而非Cell对象，不能再用`.value`属性
**解决**：直接使用`list(ws.iter_rows(...))[0]`获取元组作为headers

### 3.2 数据结构问题

**问题：前端store访问数据结构错误**
```
表现：历史成绩列表显示共0条记录
```
**原因**：API返回结构是`{success: true, data: {records: [...], total: N}}`，但store直接访问`res.data.records`
**解决**：修正store中数据访问路径为`res.data?.data?.records`

### 3.3 中文编码问题

**问题：导出Excel时文件名编码错误**
```
错误信息：'latin-1' codec can't encode characters in position 29-33
```
**原因**：HTTP头部的Content-Disposition不支持非ASCII字符
**解决**：对文件名进行URL编码：`filename*=UTF-8''{quote(filename, safe='')}`

### 3.4 雷达图文字叠层问题

**问题**：雷达图的轴标签与图例、标题重叠
**表现**：平时表现、期中考试等标签被遮挡，底部图例与期末考试标签重叠
**解决**：
1. 调整雷达图中心位置 `center: ['50%', '52%']`
2. 缩小雷达图半径 `radius: '60%'`
3. 增加图表容器高度至400px
4. 为轴标签添加足够padding
5. 图例放在底部水平排列，增加间距

### 3.5 最低分显示问题

**问题**：当min_score为0时，显示"0"但前端用`||`导致显示"-"
**解决**：使用严格比较 `min_score != null` 替代 `||`

## 四、经验总结

### 4.1 技术经验

1. **FastAPI异步开发**
   - 使用`async/await`处理所有IO操作
   - 数据库操作通过`await db.execute()`执行
   - 注意避免在异步函数中使用同步的阻塞操作

2. **openpyxl Excel处理**
   - `iter_rows(min_row=x, max_row=y, values_only=True)` 返回元组列表
   - 要先确定表头行位置，再读取数据
   - 避免使用`.value`访问已经用`values_only=True`的单元格

3. **ECharts图表配置**
   - radar图需要合理设置`center`和`radius`避免与标签重叠
   - 图例和坐标轴标签需要足够的空间
   - 设置`min: 0`避免负数轴导致的刻度问题

4. **JWT认证**
   - 教师端和学生端使用不同的认证路由
   - 教师端使用`/api/auth/teacher/login`
   - 前端store和路由需要正确保存token

### 4.2 开发流程经验

1. **问题定位**
   - 先用curl测试后端API确认功能正常
   - 检查数据库记录确认数据写入
   - 前端问题通过console.log定位

2. **数据流追踪**
   - 理解API响应的嵌套结构（通常是`res.data.data`）
   - 明确store、组件、API的职责边界
   - 统一状态管理避免数据不一致

3. **用户体验优化**
   - 上传结果显示统计信息让用户快速确认
   - 图表布局要留足空间避免叠层
   - 错误提示要具体便于用户理解

### 4.3 讯飞API使用

1. **星火大模型调用**
   - 使用HTTP方式调用`chat_completion`接口
   - Prompt设计要明确输出格式（JSON）
   - 添加fallback机制，API失败时生成默认报告

2. **重试机制**
   - 讯飞API可能因网络问题失败
   - 建议添加重试逻辑（最多3次）
