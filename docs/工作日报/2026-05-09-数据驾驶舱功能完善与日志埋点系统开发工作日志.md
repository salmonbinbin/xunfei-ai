# 2026-05-09 数据驾驶舱（管理端）功能完善与日志埋点系统开发工作日志

## 一、工作概述

本次工作的核心目标是为管理端数据驾驶舱完善数据埋点体系，确保学生端/教师端的所有功能操作都能被正确记录并反映在管理端的可视化看板中。

**主要成果：**
1. 建立了完整的用户操作日志、API调用日志、登录日志服务
2. 修复了管理端数据看板的所有SQL查询问题
3. 修复了数据库表结构与ORM模型不匹配的问题
4. 实现了日志数据的异步非阻塞写入机制

### 2.1 技术栈

| 层级 | 技术 | 说明 |
|-----|------|------|
| 后端框架 | FastAPI + SQLAlchemy 2.0 + aiomysql | 异步ORM，异步数据库操作 |
| 认证 | JWT (python-jose + bcrypt) | HS256算法，用户/管理员/教师三端分离 |

## 六、遇到的问题与解决方案

### 6.1 问题一：SQLAlchemy 2.0 类型错误

**问题描述：**
```
sqlalchemy.exc.ArgumentError: Object <class 'app.models.user_log.ApiLog'> is not legal as a SQL literal value
```

**原因：**
在SQLAlchemy 2.0中，`func.count()` 不能直接传模型类作为参数，需要传 `func.count(Model.id)` 或 `func.count("*")`。

**错误代码：**
```python
# 错误 ❌
select(func.count(ApiLog))

# 正确 ✅
select(func.count(ApiLog.id))
```

### 6.2 问题二：func.cast 表达式优先级

**问题描述：**
SQLAlchemy将 `func.cast(a == b, Integer)` 误解为 `(func.cast(a)) == b`

**原因：**
比较表达式的括号位置不对

**错误代码：**
```python
# 错误 ❌
func.sum(func.cast(ApiLog.call_type == "fail", Integer))

# 正确 ✅
func.sum(cast((ApiLog.call_type == "fail"), Integer))
```

### 6.3 问题三：参数名与模块名冲突

**问题描述：**
```
AttributeError: 'NoneType' object has no attribute 'today'
```

**原因：**
路由函数参数名 `date` 与 `datetime.date` 混淆，FastAPI将 `date` 参数绑定到了 `None`。

**错误代码：**
```python
# 错误 ❌
async def get_hourly_trend(request: Request, date: Optional[str] = None, ...):
    target_date_str = date or date.today().isoformat()  # date是None，不是datetime.date

# 正确 ✅
async def get_hourly_trend(request: Request, query_date: Optional[str] = None, ...):
    target_date_str = query_date if query_date is not None else date.today().isoformat()
```

### 6.4 问题四：SQLAlchemy 2.0 text() 构造限制

**问题描述：**
```
sqlalchemy.exc.ArgumentError: Removing unpermitted intrinsics
```

**原因：**
SQLAlchemy 2.0不允许在 `text()` 构造中使用 `DISTINCT`、`HOUR` 等SQL关键字。

**错误代码：**
```python
# 错误 ❌
select(text("HOUR(created_at) as hour"))
select(func.count(text("DISTINCT user_id")))

# 正确 ✅
select(func.extract("hour", LoginLog.created_at))
select(func.count(distinct(LoginLog.user_id)))
```

### 6.5 问题五：数据库表缺少基类字段

**问题描述：**
```
pymysql.err.OperationalError: (1054, "Unknown column 'is_deleted' in 'field list'")
pymysql.err.OperationalError: (1054, "Unknown column 'updated_at' in 'field list'")
```

**原因：**
日志表在创建时只包含了模型定义的部分字段，但 `BaseModel` 基类会注入 `is_deleted` 和 `updated_at` 字段。

**解决方案：**
```sql
ALTER TABLE login_logs ADD COLUMN is_deleted TINYINT(1) DEFAULT 0;
ALTER TABLE login_logs ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
-- 对 api_logs, user_logs 同样执行
```

### 6.6 问题六：管理员密码验证失败

**问题描述：**
登录管理端返回401 "用户名或密码错误"

**原因：**
数据库中存储的密码哈希与 `admin123` 不匹配

**解决方案：**
```python
# 重置管理员密码
password_hash = bcrypt.hashpw(b'admin123', bcrypt.gensalt()).decode('utf-8')
await db.execute(update(Admin).where(Admin.id == 1).values(password=password_hash))
```

### 6.7 问题七：异步任务在FastAPI中未执行

**问题描述：**
`asyncio.create_task()` 创建的任务似乎没有执行

**原因：**
需要确保事件循环正在运行。在FastAPI的 `async def` 函数中，直接调用 `asyncio.create_task()` 即可。

**正确用法：**
```python
@router.post("/something")
async def endpoint():
    try:
        asyncio.create_task(save_log(...))  # ✅ 直接创建
    except Exception as e:
        logger.warning(f"Log save failed: {e}")  # 静默处理，不阻塞
    return {"success": True}
```

## 八、经验总结

### 8.1 异步日志处理

1. 使用 `asyncio.create_task()` 实现非阻塞日志写入
2. 用 `try/except` 包裹，防止日志写入失败影响主业务
3. 日志服务本身的异常不向上传播，静默记录到logger

### 8.2 SQLAlchemy 2.0 迁移要点

1. `func.count(Model)` → `func.count(Model.id)`
2. `text()` 构造尽量少用，优先使用 SQLAlchemy 表达式
3. `extract()` 函数替代 `text("HOUR(...)")` 等
4. 比较表达式需要括号 `cast((expr), type)`

### 8.3 日志设计原则

1. **非阻塞**：异步写入，不影响API响应时间
2. **可容错**：写入失败不影响主业务
3. **可追溯**：记录足够的上下文信息（用户、时间、IP、操作）
4. **可统计**：字段设计支持分组、计数、去重

### 8.4 数据驾驶舱计数公式

| 指标 | 计算公式 |
|-----|---------|
| 今日在线人数 | `COUNT(DISTINCT user_id) FROM login_logs WHERE DATE(created_at) = TODAY` |
| 今日登录学生数 | 同上 + `AND user_type = 'student'` |
| 今日API调用量 | `COUNT(*) FROM api_logs WHERE DATE(created_at) = TODAY` |
| 功能使用排行 | `COUNT(*) FROM user_logs GROUP BY module ORDER BY COUNT(*) DESC LIMIT 5` |
| API失败率 | `SUM(call_type='fail') / COUNT(*) * 100` |
| 每小时用户数 | `COUNT(DISTINCT user_id) FROM login_logs GROUP BY HOUR(created_at)` |

### 8.5 数据库Schema管理

1. ORM模型定义的字段必须与数据库实际表结构一致
2. 使用 `BaseModel` 基类时，确保所有表都有 `is_deleted` 和 `updated_at`
3. 数据库迁移脚本应该同步更新ORM模型和实际表结构
