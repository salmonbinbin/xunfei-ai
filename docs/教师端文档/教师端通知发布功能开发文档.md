# 教师端智能通知发布功能 - 开发文档

> 本文档定义智能通知发布功能的开发规范、技术实现、API设计。
> 基于学生端宣传文案生成功能（Activity.vue）的成功模式设计。

---

## 一、功能概述

### 1.1 产品定位

**"AI小商教师端，智能通知发布"**

面向广州商学院教师，通过AI能力辅助生成格式规范的通知文档，支持一键发送到飞书群。

### 1.2 核心价值

- **省时**：输入主题+补充信息，AI一键生成规范通知
- **规范**：格式统一，语气正式，符合校园通知标准
- **高效**：一键复制 + 飞书群推送

---

## 二、功能需求

### 2.1 通知类型

| 类型 | 标识 | 说明 |
|-----|------|------|
| 考试通知 | exam | 期末/期中考试安排 |
| 会议通知 | meeting | 教师/班级会议 |
| 活动通知 | activity | 校园活动 |
| 放假通知 | holiday | 节假日放假安排 |
| 征稿通知 | submission | 征文/投稿征集 |
| 其他通知 | other | 其他类型 |

### 2.2 输入字段

| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 通知类型 | 下拉选择 | 是 | 如上表 |
| 通知主题 | 文本输入 | 是 | 例如"关于期末考试安排的通知" |
| 补充信息 | 文本输入 | 否 | 考试时间、地点、会议议程等 |

### 2.3 输出内容

AI生成的通知包含以下部分：
- **标题**：`【通知】+ 主题`
- **称呼**：尊敬的各位同学/老师
- **正文**：分点列出，逻辑清晰
- **落款**：广州商学院XX学院 + 日期

## 四、API设计

### 4.1 生成通知

```
POST /api/teacher/notification/generate
```

**请求体**：
```json
{
  "notification_type": "exam",
  "topic": "关于2026年春季学期期末考试安排的通知",
  "additional_info": "考试时间：6月15日上午 考试地点：9教501-503"
}
```

**响应**：
```json
{
  "success": true,
  "data": {
    "title": "【通知】关于2026年春季学期期末考试安排的通知",
    "content": "尊敬的各位同学：\n\n关于2026年春季学期期末考试安排的通知...\n\n广州商学院经济管理学院\n2026年5月10日",
    "raw_response": "..."
  }
}
```

### 4.2 发送飞书

```
POST /api/teacher/notification/send
```

**请求体**：
```json
{
  "title": "【通知】关于期末考试安排的通知",
  "content": "尊敬的各位同学：\n\n...",
  "group_id": "student_union"
}
```

**响应**：
```json
{
  "success": true,
  "message": "消息发送成功"
}
```

### 4.3 获取可用群组

```
GET /api/teacher/notification/groups
```

**响应**：
```json
{
  "success": true,
  "data": [
    {"group_id": "student_union", "name": "学生会通知"},
    {"group_id": "club_alliance", "name": "社团联盟"},
    {"group_id": "class_group", "name": "班级通知"},
    {"group_id": "youth_union", "name": "团委通知"},
    {"group_id": "dormitory", "name": "宿舍管理"}
  ]
}
```

## 五、数据库设计（可选）

> 当前版本为MVP实现，不持久化存储通知记录。如需后续扩展通知历史功能，可创建以下表。

### 5.1 表结构（可选扩展）

```sql
CREATE TABLE teacher_notifications (
    id INT PRIMARY KEY AUTO_INCREMENT,
    teacher_id INT NOT NULL,
    notification_type VARCHAR(50) NOT NULL,
    title VARCHAR(200),
    content TEXT,
    additional_info TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (teacher_id) REFERENCES users(id)
);
```

### 5.2 说明

- `teacher_id`：关联教师用户
- `notification_type`：通知类型（exam/meeting/activity/holiday/submission/other）
- `title`：生成的标题
- `content`：生成的正文
- `additional_info`：用户输入的补充信息
- `created_at`：创建时间

        <el-button type="primary" @click="sendToFeishu">确认发送</el-button>

## 八、状态流转

```
空闲 → 输入信息 → 点击生成 → AI处理中(loading)
                                      ↓
                          生成完成 → 可复制/可推送
                              ↓
                          推送失败 → 显示错误 → 可重试
```
