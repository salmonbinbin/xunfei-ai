# AI小商 - 部署运维指南

> 环境搭建、Docker部署、运维监控。

---

## 一、环境要求

| 软件 | 版本 | 说明 |
|-----|-----|-----|
| Python | 3.10+ | 后端运行环境 |
| Node.js | 18+ | 前端构建 |
| MySQL | 8.0 | 数据库 |
| Docker | 24.x | 容器化部署 |
| Nginx | 1.25+ | 反向代理 |

## 二、本地开发环境

### 2.1 后端

```bash
cd backend

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置环境变量
cp ../.env.example .env
# 编辑 .env 填入数据库和讯飞API信息

# 初始化数据库
python scripts/init_db.py

# 启动开发服务器
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 2.2 前端

```bash
cd frontend

# 安装依赖
npm install

# 配置环境变量
cp .env.example .env.production
# 编辑 .env.production 填入API地址

# 启动开发服务器
npm run dev
```

## 三、Docker 部署

### 3.1 docker-compose.yml

```yaml
version: '3.8'

services:
  # MySQL数据库
  db:
    image: mysql:8.0
    container_name: ai_xiaoshang_db
    environment:
      MYSQL_ROOT_PASSWORD: root_password_change_me
      MYSQL_DATABASE: ai_xiaoshang
      MYSQL_USER: ai_user
      MYSQL_PASSWORD: ai_password_change_me
    volumes:
      - mysql_data:/var/lib/mysql
      - ./backend/scripts/init_db.sql:/docker-entrypoint-initdb.d/init.sql:ro
    ports:
      - "3306:3306"
    networks:
      - ai_xiaoshang
    restart: unless-stopped

  # 后端API服务
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: ai_xiaoshang_backend
    environment:
      - DATABASE_URL=mysql+aiomysql://ai_user:ai_password_change_me@db:3306/ai_xiaoshang
      - XFYUN_APP_ID=${XFYUN_APP_ID}
      - XFYUN_API_KEY=${XFYUN_API_KEY}
      - XFYUN_API_SECRET=${XFYUN_API_SECRET}
      - SECRET_KEY=${SECRET_KEY}
    depends_on:
      - db
    networks:
      - ai_xiaoshang
    restart: unless-stopped
    volumes:
      - ./uploads:/app/uploads

  # 前端Nginx服务
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: ai_xiaoshang_frontend
    depends_on:
      - backend
    networks:
      - ai_xiaoshang
    restart: unless-stopped

networks:
  ai_xiaoshang:
    driver: bridge

volumes:
  mysql_data:
```

### 3.2 启动命令

```bash
# 复制环境变量模板
cp .env.example .env
# 编辑 .env 填入实际值

# 启动所有服务
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

## 五、Nginx 配置

```nginx
# nginx/nginx.conf

server {
    listen 80;
    server_name your-domain.com;

    # 前端静态资源
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # 后端API代理
    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # WebSocket支持
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 86400;
    }

    # 上传文件（私有）
    location /uploads {
        alias /var/www/uploads;
        internal;
    }
}
```

## 六、运维检查清单

### 6.1 启动后检查

```bash

# 检查日志
```

## 八、常见问题

| 问题 | 解决方案 |
|-----|---------|
| 后端启动失败 | 检查数据库连接和讯飞API配置 |
| 前端404 | 检查Nginx配置和build结果 |
| 讯飞API超时 | 增加重试次数，检查网络 |
| 文件上传失败 | 检查uploads目录权限 |
