# AI小商 - 接口规范

> API接口定义、请求响应格式。详细业务逻辑见《产品需求PRD.md》。

---

## 一、接口概览

| 模块 | 前缀 | 方法 | 说明 |
|-----|-----|-----|-----|
| 认证 | `/api/auth` | POST | 登录、Token刷新 |
| 课表 | `/api/timetable` | GET/POST | 获取课表、OCR导入 |
| 日程 | `/api/schedule` | GET/POST/DELETE | 日程CRUD |
| 录音回顾 | `/api/review` | POST/GET | 上传录音、获取列表 |
| 回顾详情 | `/api/review/{id}` | GET | 获取回顾详情 |
| AI对话 | `/api/chat` | POST | 发送消息(流式) |
| 翻译 | `/api/translate` | POST | 即时翻译 |
| 翻译 | `/api/translate/docx` | POST | 文档翻译 |
| 翻译 | `/api/translate/preview/{id}` | GET | 获取翻译预览 |
| 选课助手 | `/api/course/advisor` | POST | 获取推荐 |
| 活动助手 | `/api/activity` | POST | 生成文案 |
| 导出 | `/api/export/docx` | POST | 导出DOCX |
| 导出 | `/api/export/pptx` | POST | 导出PPTX |
| 管理-知识库 | `/api/admin/knowledge` | GET/POST/PUT/DELETE | 知识库CRUD |
| 管理-看板 | `/api/admin/dashboard` | GET | 统计数据 |
| 管理-用户 | `/api/admin/users` | GET/POST | 用户管理 |

## 六、AI对话

### 6.1 POST /api/chat/message

**请求：**
```json
{
  "conv_id": 1,
  "message": "我想考公务员，下学期选什么课好？",
  "mode": "normal",
  "stream": true
}
```

**响应（SSE流式）：**
```
data: {"type": "content", "content": "根据"}
data: {"type": "content", "content": "你的"}
data: {"type": "content", "content": "经济学"}
...
data: {"type": "complete", "sentiment": "neutral", "suggest_emotion": false}
```

**非流式响应：**
```json
{
  "success": true,
  "data": {
    "reply": "根据你的经济学背景...",
    "conv_id": 1,
    "sentiment": "neutral"
  }
}
```

### 6.2 POST /api/chat/voice

**请求：** `multipart/form-data`
```
audio: [文件]
mode: normal/emotion
```

**响应：**
```json
{
  "success": true,
  "data": {
    "recognized_text": "最近心情不好",
    "reply_text": "听起来你有些难过...",
    "voice_url": "/uploads/voice/1.mp3",
    "sentiment": "negative",
    "suggest_emotion": true
  }
}
```

## 七、智能翻译

### 7.1 POST /api/translate

**即时翻译接口**

**请求：**
```json
{
  "text": "Hello, how are you?",
  "source_lang": "en",
  "target_lang": "zh"
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "translated_text": "你好，你怎么样？",
    "source_lang": "en",
    "target_lang": "zh"
  }
}
```

**支持语言代码：**
| 代码 | 语言 |
|-----|------|
| zh | 中文 |
| en | 英文 |
| ja | 日文 |
| ko | 韩文 |
| fr | 法文 |
| de | 德文 |
| es | 西班牙文 |
| ru | 俄文 |
| ar | 阿拉伯文 |

### 7.2 POST /api/translate/docx

**文档翻译接口**

**请求：** `multipart/form-data`
```
file: [文件]  # .txt 或 .docx 文件
target_lang: zh  # 目标语言代码
```

**响应：**
```json
{
  "success": true,
  "data": {
    "task_id": "trans_123456",
    "message": "文档上传成功，正在翻译中...",
    "status": "processing"
  }
}
```

### 7.3 GET /api/translate/preview/{task_id}

**获取翻译预览**

**响应：**
```json
{
  "success": true,
  "data": {
    "task_id": "trans_123456",
    "status": "completed",
    "translated_content": "这是翻译后的文档内容...",
    "word_count": 1500,
    "created_at": "2026-04-22T10:00:00"
  }
}
```

**状态说明：**
| 状态 | 说明 |
|-----|------|
| processing | 翻译中 |
| completed | 翻译完成 |
| failed | 翻译失败 |

### 7.4 GET /api/translate/download/{task_id}

**下载翻译后的Word文档**

**响应：** 文件流（.docx）

## 十一、错误码

| 错误码 | 说明 |
|-------|-----|
| NOT_FOUND | 资源不存在 |
| VALIDATION_ERROR | 数据验证失败 |
| THIRD_PARTY_ERROR | 第三方服务错误 |
| AUTH_ERROR | 认证失败 |
| FORBIDDEN | 无权限 |
| INTERNAL_ERROR | 服务器内部错误 |
