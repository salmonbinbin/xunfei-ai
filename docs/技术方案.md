## 四、讯飞API集成详解

### 4.1 讯飞能力调用流程

```
用户请求
    ↓
后端服务（FastAPI）
    ↓
讯飞API（REST/WebSocket）
    ↓
返回结果
```

### 4.2 鉴权机制

讯飞API使用 HMAC-SHA256 签名鉴权，核心流程：

```python
# 1. 构建待签名字符串
signature_origin = f"host: {host}\ndate: {date}\nPOST {path} HTTP/1.1"

# 2. HMAC-SHA256签名
signature_sha = hmac.new(
    api_secret.encode('utf-8'),
    signature_origin.encode('utf-8'),
    digestmod=hashlib.sha256
).digest()

# 3. Base64编码
authorization = base64.b64encode(signature_sha).decode('utf-8')
```

### 4.3 重试机制

所有讯飞API调用必须实现3次重试，间隔递增：

```python
for attempt in range(3):
    try:
        result = await call_xfyun_api()
        return result
    except Exception as e:
        if attempt < 2:
            await asyncio.sleep(1.0 * (attempt + 1))
        else:
            raise ThirdPartyException("Service name", str(e))
```

## 五、核心业务流程

### 5.1 课表OCR导入

```
用户上传课表图片
    ↓
后端接收 → 保存临时文件
    ↓
调用讯飞OCR API → 获取图片文字
    ↓
调用星火大模型 → 解析为结构化课程数据
    ↓
用户确认 → 写入MySQL courses表
```

### 5.2 AI学姐对话

```
用户发送消息（文字/语音）
    ↓
语音 → 讯飞ASR → 转文字
    ↓
文本 → 讯飞NLP → 情感分析
    ↓
文本 → ChromaDB RAG → 检索相关知识
    ↓
构建提示词 → 讯飞星火大模型 → 生成回复
    ↓
文本回复 + 讯飞TTS（可选） → 返回前端
```

### 5.3 录音回顾

```
用户上传MP3文件
    ↓
后端接收 → 保存文件
    ↓
调用讯飞ASR（录音文件识别）→ 轮询获取转写结果
    ↓
转写文字 → 讯飞星火大模型 → 生成结构化总结
    ↓
写入MySQL summaries表
    ↓
用户可导出：DOCX（python-docx）/ PPT（python-pptx）
```
